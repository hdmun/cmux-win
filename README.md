# cmux-win
WinUI-based Windows terminal with vertical tabs and notifications for AI coding agents

## 🤖 AI 에이전트 개발 파이프라인 가이드

이 리포지토리는 일반적인 개발 환경과 달리, AI 에이전트(Claude, Copilot 등)가 자율적으로 기획을 읽고 코드를 구현하도록 설계된 독특한 폴더 구조와 파이프라인을 가집니다.

### 1. 핵심 폴더 구조 및 역할

AI 에이전트는 아래 세 폴더를 **"설계도 - 상태 관리 - 작업 규칙"**으로 분리하여 활용합니다.

- 📂 **`_workspace` (설계 및 아키텍처)**
  - **역할**: 사람과 AI가 읽고 참조하는 **"설계도"** 및 스펙 문서입니다.
  - **특징**: 기능 명세, 아키텍처 다이어그램 등 목표 상태(Source of Truth)를 정의하며, 실제 소스 코드는 절대 추가하지 않습니다.
- 📂 **`plans` (실행 상태 및 체크리스트)**
  - **역할**: 기계(AI)가 판독 가능한 **"상태 및 진행표"**입니다.
  - **특징**: 마일스톤, 태스크 상태, 참조해야 할 설계 문서 링크(`doc_refs`) 등 실행 메타데이터를 JSON 형태로 관리합니다.
- 📂 **`.rules` (작업 규약 및 제약 조건)**
  - **역할**: AI 에이전트가 반드시 지켜야 하는 **"법전(작업 규칙)"**입니다.
  - **특징**: 워크플로우 진행 방식, 수정 금지 경로, 문서 동기화 규칙 등 에이전트의 행동 강령이 명시되어 있습니다. (전체 인덱스는 루트의 `AGENTS.md` 참조)

> **상관관계 요약**: AI 에이전트는 **`.rules`**가 정해준 규칙 안에서, **`plans`**의 일정표에 따라, **`_workspace`**의 설계도를 참조하여 실제 코드를 구현합니다.

---

### 2. AI 에이전트 자율 실행 방법 (Autonomous Execution)

이 리포지토리는 `cmux-plan` 파이썬 CLI 스크립트를 통해 에이전트의 개발 루프를 자동화합니다. 사용자는 에이전트에게 **"다음 태스크 이어서 진행해 줘"** 또는 **"자율 실행 계속해 줘"**라고 지시하기만 하면 됩니다.

에이전트는 내부적으로 다음 7단계 루프를 자율적으로 실행합니다:

1. **작업 탐색 (`next`)**: 의존성이 해결된 다음 진행 가능한 작업을 찾습니다.
2. **작업 시작 (`status in_progress`)**: 작업을 할당받고 상태를 진행 중으로 변경합니다.
3. **작업 브리핑 (`brief`)**: 방대한 문서를 다 읽지 않고, 해당 작업에 필요한 규칙과 설계 문서 요약본만 추출하여 읽습니다.
4. **코드 구현**: 브리핑을 바탕으로 `src/`, `cli/` 등의 허용된 경로에 코드를 작성합니다.
5. **자동 검증 (`verify`)**: 빌드 및 테스트 명령어를 실행하여 코드가 올바른지 기계적으로 검증합니다.
6. **무결성 확인 (`validate`)**: 스키마 오류나 문서 간 불일치(Drift)가 없는지 전체를 확인합니다 (0 Error 필요).
7. **작업 완료 (`status done`)**: 검증 통과 후 작업을 완료 처리하고 다음 마일스톤 승급 및 세션 인수인계를 진행합니다.

상세한 권한 모델과 스크립트 사용법은 `_workspace/13-autonomous-execution.md` 및 `.claude/skills/cmux-win-autonomous-execution/SKILL.md`를 참조하세요.
